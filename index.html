    • Introduction

    Information

    Base URL : https://nipa-ticket-api.herokuapp.com
    Endpoints
    Method Endpoint
    GET /ticket/all /ticket/filter/{status} /ticket/get/{id}
    POST /ticket/create
    PUT /ticket/update

    HTTP Response

  • 200 - OK, The request was successful
  • 404 - Not Found, Resource was not found

    • JSON Error Response

    This is a typical error response
    {
       "status" : false,
       "message" : "Invalid information",
       "tickets" : null,
    }


    Reference

    All tickets

    Description: Request all tickets information and sort by id (default), status and latest_update_timestamp (optional)
    Method: GET
    Endpoint: /ticket/all
    Parameters (Optional)
    Parameter Argument Description
    status_sort asc desc Sort tickets by status order by ascending or descending
    update_sort asc desc Sort tickets by latest_update_timestamp order by ascending or descending

    Specific status tickets

    Description: Request specific status tickets information and sort by id (default), latest_update_timestamp (optional)
    Method: GET
    Endpoint: /ticket/filter/{status}
  • status : specific status tickets (pending, accepted, solved, rejected)
    • Parameters (Optional)
    Parameter Argument Description
    sort asc desc Sort specific status tickets by latest_update_timestamp order by ascending or descending

    Specific id ticket

    Description: Request specific id ticket information
    Method: GET
    Endpoint: /ticket/get/{id}
  • id : specific id tickets (integer)

    • Create ticket

    Description: Request to create a new ticket
    Method: POST
    Endpoint: /ticket/create
    Parameters (Essential)
    Parameter Argument Description
    title String Ticket title
    description String Ticket description
    name String Requester full name
    tel String (telephone number) Requester telephone number
    email String (email) Requester email
    channel Facebook Line Web Form Contact channel to reply

    Update ticket

    Description: Update specific id ticket information (can update partial information)
    Method: PUT
    Endpoint: /ticket/update
    Parameters
    Parameter Argument Description
    id (Essential) Integer Sort specific status tickets by latest_update_timestamp order by ascending or descending
    title (Optional) String Ticket title
    description (Optional) String Ticket description
    name (Optional) String Requester name
    tel (Optional) String (telephone number) Requester telephone number
    email (Optional) String (email) Requester email
    channel (Optional) Facebook Line Web Form Contact channel to reply
    status (Optional) pending accepted resolved rejected Status of ticket
